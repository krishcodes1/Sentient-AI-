Security Architecture
Four layers of prompt injection defense.
Layer 1
Input Sanitization
Strips hidden HTML, zero-width chars, homoglyph attacks, and instruction-like phrases from all external content.
Layer 2
LLM Classifier
A secondary, locally-hosted LLM evaluates content for semantic injection attempts with confidence scoring.
Layer 3
Output Validation
Every agent action is validated against permission policy before execution. Exceeding scope = hard block.
Layer 4
Behavioral Anomaly
Real-time monitoring tracks agent patterns and flags deviations from expected tool usage behavior.

Permission Engine
Tiered trust. Zero implicit access.
Auto-Approve
Read-only operations on approved connectors. Fetching grades, reading calendar events, viewing portfolio data.
User-Confirm
Write operations that modify external state. Sending emails, creating calendar events, submitting assignments.
Admin-Only
Sensitive configuration changes. Adding new connectors, modifying permission policies, managing API keys.
Hard-Blocked
Permanently forbidden. Financial transactions, credential exfiltration, privilege escalation. Cannot be overridden.

How It Works
From signup to secure AI chat.
1. Create Account & Onboard
Register with email + password. The onboarding wizard walks you through picking an AI provider, entering your API key, and choosing a model.
2. Backend Writes Config
Your settings are encrypted and written to openclaw.json on a shared Docker volume. The OpenClaw gateway picks it up automatically.
3. Connect Channels
Add Telegram bots, Discord bots, Slack apps, or WhatsApp via QR scan. Tokens are AES-256-GCM encrypted in PostgreSQL.
4. Chat Securely
Messages flow through the gateway, pass injection defense layers, hit your AI provider, and return validated responses. Every action is audit-logged.
Honest Assessment
Challenges & limitations.
LLM Inference Overhead
Multi-layer security adds latency. Local models via Ollama require GPU hardware for acceptable speeds.
Robinhood API Gaps
No public stock API exists. Only official Crypto API is integrated. Expansion depends on Robinhood's decisions.
Canvas Institutional Deps
Requires admin-issued developer keys. Not a plug-and-play solution without university IT coordination.
Academic Scope
Single-semester project. Core architecture is demonstrated but full adversarial testing is beyond timeline.
